# Security Policy

## Supported Versions

Currently, only the latest release of Flux-Titan is supported with security updates. We recommend always running the latest version from the `main` branch or the latest tagged release.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

We take the security of Flux-Titan seriously. If you believe you have found a security vulnerability in this project, **please do not report it through public GitHub issues.**

Instead, please report it to us by:
1. Sending a private email to the project maintainers.
2. Opening a draft security advisory via GitHub's Security tab (if enabled for the repository).

Please include the following information in your report:
* Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
* Full paths of source file(s) related to the manifestation of the issue
* The location of the affected source code (tag/branch/commit or direct URL)
* Any special configuration required to reproduce the issue
* Step-by-step instructions to reproduce the issue
* Proof-of-concept or exploit code (if possible)
* Impact of the issue, including how an attacker might exploit the issue

We will prioritize the issue, evaluate the impact, and work on a fix as quickly as possible. We will endeavor to acknowledge receipt of your vulnerability report within 48 hours.

Thank you for helping to keep Flux-Titan and our users safe!
